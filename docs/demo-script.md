# Academic-Nexus 答辩演示脚本

## 1. 管理员登录

- 使用 `admin001 / 123456` 登录。
- 说明系统分为学生、教师、管理员三类角色，菜单由后端权限返回。

## 2. 系统健康中心

- 打开 `/admin/system-health`。
- 展示 MySQL、Redis、AI Service / Ollama、上传目录、Flyway、JVM。
- 说明 DEGRADED 表示可兜底运行，DOWN 表示需要处理。

## 3. 审计中心

- 打开 `/admin/audit-logs`。
- 演示模块筛选、风险筛选、失败原因和 traceId。
- 说明成绩、权限、AI SQL、Redis 修复等高风险操作都会留痕。

## 4. 权限矩阵

- 打开 `/admin/permission-matrix`。
- 展示学生、教师、管理员分别拥有的菜单和能力权限。
- 说明前端隐藏菜单不是安全边界，后端接口仍会做角色和权限拦截。

## 5. 三角色班级闭环

- 管理员打开班级管理，为班级绑定班主任账号。
- 学生登录后打开 `/student/class`，查看本人班级、班主任和同班同学概要。
- 教师登录后打开 `/teacher/classes`，查看自己负责的班级和学生名单。
- 说明学生名单只展示学号、姓名和状态，不暴露联系方式等隐私字段。

## 6. 学生选课

- 使用 `student001 / 123456` 登录。
- 打开选课页面，展示容量、已选、窗口状态。
- 点击选课后再重复点击，演示重复选课拦截。

## 7. Redis 一致性报告

- 管理员打开 `/admin/course-selection-consistency`。
- 点击手动校验，展示 Redis 库存、数据库已选人数、理论库存、差值。
- 如 Redis 库存不一致，演示二次确认修复。

## 8. 教师成绩维护

- 使用 `teacher001 / 123456` 登录。
- 打开教师成绩维护页，展示教学班成绩录入。
- 说明已发布、锁定、修改原因和审计是成绩敏感操作的保护重点。

## 9. 学生成绩查询

- 学生打开成绩查询页。
- 展示按学期查看成绩、绩点和考试类型。

## 10. 学业进度

- 学生打开学业画像或学业进度入口。
- 展示已修课程、毕业审核、培养方案和风险说明。

## 11. AI 教务助手

- 打开 AI 教务助手。
- 提问“选课失败可能是什么原因”。
- 展示证据来源、可信度、兜底状态和 traceId。

## 12. AI 模型与安全审查

- 管理员打开 AI 模型管理页，展示 Qwen3、qwen2.5-coder、Qwythos-9B 等模型预设。
- 展示搜索配置、安全策略和命中日志。
- 输入涉及成绩、密码、密钥或个人隐私的问题，说明系统会拦截并记录。

## 13. Spring Cloud 服务调用

- 启动 Nacos、AI 服务和主系统。
- 打开 Nacos 控制台，展示 `academic-main` 和 `academic-ai-service`。
- 说明主系统通过 OpenFeign + LoadBalancer 按服务名调用 AI 服务。
- Ollama 未启用时，AI 服务仍提供可演示的本地兜底响应。

## 14. 数据库安全查询

- 管理员打开 `/admin/ai-sql`。
- 输入自然语言生成 SQL，展示 SELECT only、安全检查、确认执行。
- 说明敏感字段和高风险 SQL 会被拦截。

## 15. 压测报告

- 打开 `/admin/load-test-reports`。
- 展示抢课成功数、满员数、吞吐量、平均延迟、P95 和 Redis 状态。
