## 7. AI Model Delete Verification

Covered by `AiModelDeleteRegressionTests`:

- Admin can soft-delete disabled non-default model.
- Deleted model is hidden from active model list.
- Historical `ai_call_log` records remain readable.
- Default model cannot be deleted.
- Enabled model cannot be deleted.
- Student cannot delete model.
- Delete operation writes audit.

## 8. Search And Safety Verification

Covered by `AiSearchSafetyConfigTemplateTests`:

- Search templates can be read.
- Local demo search template can be applied through existing config update.
- Search test returns sample results.
- Sensitive search query returns readable blocked message.
- Safety templates can be read.
- Strict mode blocks high-risk sample content.
- Log-only mode records without blocking.
- Student cannot test admin safety config.

## 9. Dashboard Role Isolation

Covered by `DashboardRoleScopeRegressionTests`:

- Student dashboard uses personal cards and does not show management permission cards.
- Different student accounts can show different personal values.
- Teacher dashboard counts only the teacher's own offerings and exams.
- Admin dashboard shows global/system cards.
- User without student/teacher/admin role receives a clear forbidden response.
