## 3. #109 SearXNG Docker Configuration

Result: completed in docs.

- Added `.env` copy commands for Windows and Linux/macOS.
- Added `OLLAMA_ENABLED=true`, `OLLAMA_BASE_URL`, model variables, force-recreate command, environment variable verification, host Ollama verification, container Ollama verification, and model pull commands.

## 5. #111 Web Search Grounding

Result: completed in code and tests.

Implementation:

- Chat search now runs before model generation.
- Search results are compressed into a structured grounding prompt.
- Deterministic first-result requests can be answered directly from search results.
- The answer still includes `联网搜索参考` for traceability.
- Search grounding no longer hard-codes `/think` or `/no_think`; thinking mode is applied by `thinkingMode`.

Tests:

- `WebSearchGroundingTests`

## 6. #112 AI Model Registry State

Result: completed in migration, service logic, frontend, and tests.

Implementation:

- Added `V18__v201_ai_model_registry_state_constraints.sql`.
- Disabled or deleted models cannot remain default.
- Same model type keeps at most one undeleted default model after migration cleanup.
- Default model deletion returns conflict with a clear message.
- Non-default model soft delete sets `deleted=true`, `enabled=false`, and `is_default=false`.
- Disabling a default model is rejected until another same-type enabled model is set as default.
- Frontend delete action is disabled only for default models.

Tests:

- Updated `AiModelDeleteRegressionTests`
- Added `AiModelRegistryStateTests`

## 7. #114 Frontend Interaction Closure

Result: completed.

Top global search:

- Replaced static search block with real autocomplete.
- Search suggestions come from current user menu routes only.
- Supports aliases such as course, grade, exam, timetable, users, notices, system health, teacher offerings, teacher grades, and evaluations.
- Enter navigates to the best match.
- Empty matches show `未找到相关功能`.

Dashboard quick apps:

- Settings icon is now an interactive button.
- Dialog lists current user accessible menu items.
- Supports selecting up to 8 quick apps.
- Saves to `academic-nexus:quick-apps:{username}`.
- Supports restore default.

## 8. #115 AI Chat Frontend UX

Result: completed.

Implementation:

- Added status bar showing current model, source, web search state, safety state, thinking mode, and session.
- Added sending, waiting, thinking, and typing feedback.
- Enter sends and Shift+Enter creates a new line.
- Failed send restores user input.
- Added copy answer, regenerate last answer, and clear input.
- Added error message classification for ai-service unavailable, model unavailable, Ollama unavailable, search config failure, safety block, timeout, backend error, and thinking mode unsupported.

## 9. AI Thinking Mode

Result: completed.

Implementation:

- Added `thinkingMode=AUTO|ON|OFF` to frontend, main service DTO, OpenFeign/manual HTTP payloads, and ai-service DTO.
- ai-service injects `/think` or `/no_think` only for supported models and does not duplicate explicit user directives.
- Added `thinking_mode` to chat message history with `V19__v201_ai_chat_message_thinking_mode.sql`.
- Response returns `thinkingMode`.

Tests:

- `AiThinkingModeTests`
