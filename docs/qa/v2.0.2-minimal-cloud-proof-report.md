## Summary

v2.0.2 adds a minimal, screenshotable Spring Cloud proof layer while keeping the core academic system stable. The implementation uses real services and real endpoints:

| Requirement | Evidence | Status |
| --- | --- | --- |
| Nacos discovery | `academic-main`, `academic-ai-service`, `edunexus-gateway` register in Compose | Implemented |
| Gateway routing | `edunexus-gateway:9000` routes `/api/**` to `lb://academic-main` | Implemented |
| Sentinel flow control | `@SentinelResource` on real `/api/auth/login`, QPS default 3 | Implemented |
| Sentinel dynamic rule update | `/api/cloud-proof/sentinel/login-rule` can view and update login QPS | Implemented |
| OpenFeign proof | `/api/cloud-proof/feign/ai-status` calls `academic-ai-service` | Implemented |
| Seata proof | `/api/cloud-proof/seata/commit` and `/rollback` use two demo tables | Implemented |
| Core business boundary | Seata is not forced into selection, grades, exams or applications | Preserved |
